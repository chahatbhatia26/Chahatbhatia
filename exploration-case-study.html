Personal exploration / Motion study

Flow Fragments

Exploring motion as feedback in UI.

A short study on how transitions and continuity can make state changes easier to follow, without turning the interface into performance for its own sake.

Format Personal project
Timeline 2 weeks
Focus Motion feedback and UI continuity

Quick snapshot

The work in one minute.

Problem

State changes were readable, but not always easy to mentally track.

Focus

Use motion to support comprehension instead of decoration.

Approach

Fast UI experiments comparing timing, continuity, and recovery cues.

Outcome

The strongest concepts made relationship clearer without adding noise.

Problem

Good motion is easy to overestimate.

I wanted to test where motion genuinely helps UX and where it just adds gloss. The challenge was making changes between states feel easier to understand without slowing users down.

  • State shifts sometimes felt abrupt even when the UI itself was clean.
  • Transitions looked polished, but not all of them improved comprehension.
  • The goal was support, not spectacle.

What changed

Three directions that felt worth keeping.

State continuity

Primary elements stayed connected instead of resetting between screens.

Feedback timing

Shorter, calmer transitions made feedback easier to notice and easier to trust.

Hierarchy first

Motion worked best when it reinforced hierarchy that was already clear.

Before / After

A simpler comparison.

Takeaway

Motion earns its place when it explains change, not when it performs it.
